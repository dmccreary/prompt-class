    """
Algorithm to Predict Software Project Success Using Bayesian Probability

This algorithm calculates the probability that a software project will deliver on time 
and meet all functional and non-functional requirements using Bayesian probability.

Steps:
1. Define prior probabilities for project success and failure.
2. Define likelihoods for various influencing factors given success or failure.
3. Collect evidence based on the current project's characteristics.
4. Compute the posterior probability of success given the evidence.
"""

# Prior probabilities
P_Success = 0.5
P_Failure = 1 - P_Success

# Likelihoods: P(Evidence | Success) and P(Evidence | Failure)
likelihoods = {
    'TeamExperience': {
        'Experienced': {'Success': 0.9, 'Failure': 0.1},
        'Inexperienced': {'Success': 0.2, 'Failure': 0.8},
    },
    'ProjectComplexity': {
        'High': {'Success': 0.3, 'Failure': 0.7},
        'Low': {'Success': 0.8, 'Failure': 0.2},
    },
    'RequirementsClarity': {
        'Clear': {'Success': 0.85, 'Failure': 0.15},
        'Unclear': {'Success': 0.1, 'Failure': 0.9},
    },
    'ResourceAvailability': {
        'Sufficient': {'Success': 0.8, 'Failure': 0.2},
        'Insufficient': {'Success': 0.3, 'Failure': 0.7},
    }
}

# Evidence collected for the current project
evidence = {
    'TeamExperience': 'Experienced',
    'ProjectComplexity': 'High',
    'RequirementsClarity': 'Clear',
    'ResourceAvailability': 'Sufficient'
}

# Function to compute the posterior probability of success
def compute_posterior(P_Success, P_Failure, likelihoods, evidence):
    P_Evidence_given_Success = 1
    P_Evidence_given_Failure = 1

    # Multiply the likelihoods for each piece of evidence
    for factor, value in evidence.items():
        P_Evidence_given_Success *= likelihoods[factor][value]['Success']
        P_Evidence_given_Failure *= likelihoods[factor][value]['Failure']

    # Numerator and denominator for Bayes' theorem
    numerator = P_Evidence_given_Success * P_Success
    denominator = numerator + (P_Evidence_given_Failure * P_Failure)

    # Posterior probability
    P_Success_given_Evidence = numerator / denominator
    return P_Success_given_Evidence

# Compute the posterior probability
posterior_probability = compute_posterior(P_Success, P_Failure, likelihoods, evidence)

print(f"The probability that the project will succeed is {posterior_probability * 100:.2f}%")

      • P_Failure is the complement of P_Success.

      • Define Likelihoods:

      • For each influencing factor (e.g., Team Experience, Project Complexity), we define the likelihood of observing a specific value given that the project is a success or a failure.

      • These values are based on historical data or expert judgment.

      • Collect Evidence:

      • The evidence dictionary contains the current project's characteristics.

      • For example, TeamExperience is 'Experienced', ProjectComplexity is 'High', etc.

      • Compute Posterior Probability:

      • The function compute_posterior calculates the probability of success given the evidence using Bayes' theorem.

      • It multiplies the likelihoods of each piece of evidence and combines them with the prior probabilities.
      • The final probability is normalized by dividing by the total probability of the evidence.

      • Output:

      • The posterior probability is printed, giving the likelihood that the project will succeed based on the provided evidence.

    Note:

    • This algorithm assumes independence between the influencing factors.
    • In a real-world scenario, you should obtain the likelihoods from historical project data or domain experts.
    • The prior probability can be adjusted based on organizational success rates or other relevant information.
